# cyber_physical_systems_1

# 1. Выбор начальных условий

## 1.1 Выбор набора данных

Для выполнения лабораторной работы был выбран набор данных **Vehicle Detection YOLOv8 Dataset**, размещённый на платформе Kaggle. Данный набор данных предназначен для решения задачи обнаружения объектов на изображениях и содержит примеры различных транспортных средств.

Набор данных включает следующие классы объектов:

- car
- bus
- truck
- motorcycle

Всего в наборе данных содержится более 5000 файлов, разделённых на обучающую (`train`), валидационную (`valid`) и тестовую (`test`) выборки. Разметка представлена в формате YOLO, что позволяет использовать данный набор данных напрямую в библиотеке Ultralytics и моделях семейства YOLOv11 без дополнительной конвертации.

Выбор данного набора данных обусловлен следующими причинами:

1. Набор данных имеет небольшой и удобный объём, достаточный для обучения модели в рамках лабораторной работы, но не требующий значительных вычислительных ресурсов.
2. Данные уже подготовлены и размечены в формате YOLO, что упрощает процесс запуска экспериментов.
3. Набор данных содержит несколько классов объектов, что позволяет оценить качество работы модели не только на одной категории, но и сравнить результаты для разных типов транспорта.
4. Тематика набора данных соответствует реальной практической задаче мониторинга дорожного движения.

Практическое применение выбранного набора данных может заключаться в создании системы интеллектуального транспортного мониторинга. Такая система может использоваться для:

- автоматического подсчёта количества автомобилей на дороге;
- оценки загруженности транспортных потоков;
- определения типов транспортных средств;

Подобные системы применяются в современных киберфизических системах, например в интеллектуальных транспортных сетях и системах «умного города». 

## 1.2 Выбор метрик качества

Для оценки качества работы модели были выбраны следующие метрики:

- Precision
- Recall
- mAP@0.5

### Precision

Метрика Precision показывает, какая доля объектов, найденных моделью, действительно относится к правильному классу.

Формула:

\[
Precision = \frac{TP}{TP + FP}
\]

где:
- TP — количество правильно найденных объектов;
- FP — количество ложных срабатываний.

Высокое значение Precision означает, что модель редко ошибочно определяет наличие транспортного средства там, где его нет.
Данная метрика важна для рассматриваемой задачи, поскольку ложные срабатывания могут привести к неверной оценке дорожной ситуации.

### Recall

Метрика Recall показывает, какую долю всех существующих объектов модель смогла обнаружить.

Формула:

\[
Recall = \frac{TP}{TP + FN}
\]

где:
- FN — количество пропущенных объектов.

Высокое значение Recall особенно важно для систем транспортного мониторинга, поскольку пропуск автомобиля или автобуса может привести к искажению статистики транспортного потока.

### mAP@0.5

Основной метрикой качества была выбрана mAP@0.5 (mean Average Precision при пороге IoU = 0.5).

Данная метрика является стандартной для задач обнаружения объектов и используется при оценке моделей семейства YOLO.

Метрика mAP@0.5 учитывает одновременно:
- правильность классификации объекта;
- точность определения его местоположения на изображении.

Объект считается найденным правильно, если пересечение предсказанной рамки и истинной рамки объекта (IoU) составляет не менее 0.5.

Выбор именно mAP@0.5 обусловлен тем, что:
1. Это наиболее распространённая метрика для сравнения моделей детекции объектов.
2. Она позволяет получить общую количественную оценку качества модели.
3. Метрика автоматически вычисляется библиотекой Ultralytics при обучении и тестировании модели.
